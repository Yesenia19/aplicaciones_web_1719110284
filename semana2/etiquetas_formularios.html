Etiquetas de formularios

Etiqueta Descripción Ejemplo
< input > hidden Un campo hidden se lo denomina campo oculto. Su utilidad se presenta cuando desde el servidor se genera una página dinámica por ejemplo mediante PHP y se almacena en un campo oculto un valor que se rescatará al subir el formulario al servidor. < input type="hidden" value="10:20" name="hora" >: Este parametro mandara la hora pero no la mostrara al usuario.
< input type=text > Crean campos de texto básicos de una sola línea. Se usan en cualquier lugar donde desees que el usuario ingrese un valor de una sola línea y no hay un tipo de entrada más específico disponible para recopilar ese valor. < input type="text" id="name" name="name" required minlength="4" maxlength="8" size="10" >: Esta etiqueta nos da los valores que debe llevar lo que se esta solicitando
< input type=search > Representa un campo de texto de una sola línea para propósitos de búsqueda, el cual permite a los usuarios editar una sola línea de texto regular. Los controles de búsqueda son útiles para recopilar líneas cortas de texto que los agentes procesadores o programas del lado cliente usarán como parámetros para conducir operaciones de búsqueda. Búsqueda de música: < input type="search" name="busquedamusica" >: Esta etiqueta nos ayuda a buscar algo que esta solicitando el cliente
< input type=tel > Representa un campo para un número telefónico. El control asociado a este campo es una caja de texto que permite a los usuarios editar una sola línea de texto regular, sin requerimientos particulares sobre el formato. Teléfono: < input type="tel" name="telefono" placeholder="(Código de área) Número" >
< input type=url > Representa un campo para un solo URL absoluto. El control asociado a este campo es una caja de texto que permite a los usuarios editar una sola línea de texto regular. Sitio web: < input type="url" name="sitioweb" placeholder="¿Tienes un sitio web?" >
< input type=email > Se utilizan para permitir que el usuario ingrese y edite una dirección de correo o, si se especifica el atributo multiple, una lista de direcciones de correo, editar una sola línea de texto regular. < input type="email" id="email" pattern=".+@globex.com" size="30" required >
< input type=password > Proporcionan una forma para que el usuario ingrese una contraseña de forma segura. El elemento se presenta como un control de editor de texto, sin formato, de una línea, en el que el texto está oculto para que no pueda leerse, generalmente reemplazando cada carácter con un símbolo como el asterisco ("*") o un punto ("•"). < input type="password" pattern=".{6,}" >: Este ejemplo creará un elemento password que deberá contener 6 o más carácteres.
< input type=date > Permiten a los usuarios proporcionar una fecha, bien a través de un cuadro de texto que valida automáticamente el contenido, o bien mediante una interfaz especial de selección de fechas. < input type="date" id="start" name="trip-start" value="2018-07-22" min="2018-01-01" max="2018-12-31" >
< input type=month > Representa un campo para la entrada de un mes. Mes de nacimiento: < input type="month" name="mesnacimiento" >
< input type=week > Representa un campo para la entrada de una semana. Selecciona una semana: < input type="week" name="unasemana" >
< input type=time > Representa un campo para la entrada de una hora. Tiempo estimado de arribo: < input type="time" name="eta" >
< input type=datetime-local > Representa un campo para una fecha y hora local sin información de zona horaria. Este tipo de campos es particularmente útil para recolectar un punto en el tiempo, que es específco de un lugar. Reunión: < input type="datetime-local" name="horareunion" >
< input type=number > Representa al usuario ingresar un número. Éstos incluyen validación incorporada para rechazar entradas no numéricas. El navegador puede optar por proveer flechas de pasos para permitir al usuario incrementar y decrementar el valor usando su ratón o símplemente pulsando con la punta del dedo. < input type="number" id="tentacles" name="tentacles" min="10" max="100" >
< input type=range > Permite que el usuario especificar un valor numérico comprendido entre un valor mínimo y máximo. El valor exacto, sin embargo, no se considera importante. Se repesenta típicamente como un "tirador" o un control deslizante en lugar de un campo de texto como otros tipos de < input >. < input type="range" min="5" max="10" step="0.01" >
< input type=color > Permite proporcionar a un elemento de interfaz de usuario que permite a los usuarios especificar un color, bien mediante una interfaz visual de selección, bien a través de un cuadro de texto donde ingresar un valor hexadecimal en el formato «#rrggbb». Solo se permiten colores simples (sin canal alfa). < input type="color" value="#ff0000" id="muestrario" >
< input type=checkbox > Permite insertar un vector o array de valores. El atributo value es usado para definr el valor enviado por el checkbox. El atributo checked se usa para indicar que el elemento está seleccionado. < input type="checkbox" id="cbox1" value="first_checkbox" >
< input type=radio > Permite una opción que pertenece a un grupo en el que no más de una opción puede ser seleccionada al mismo tiempo. Estos grupos están normalmente conformados por un número de botones de opción, todos compartiendo el mismo valor en el atributo name. < input type="radio" name="empleoactual" value="tiempocompleto" >
< input type=file > Permite un control para seleccionar una lista de uno o más archivos para ser subidos al servidor. < input type="file" name="archivosubido" >
< input type=submit > Permite un botón que, cuando es presionado, envía el formulario (form) al que pertenece. < input type="submit" value="Enviar datos" >
< input type=image > Representa un botón de envío gráfico, que es una imagen normal que al ser presionada, envía el formulario (form) al cual pertenece. < input type="image" name="botondeenvio" src="../../../images/submit-form.png" alt="Enviar formulario" >
< input type=reset > Representa un botón que, cuando es presionado, restaura todos los campos del formulario (form) al cual pertenece, a sus valores iniciales. < input type="reset" value="Restaurar" >
< select name="" > Indicara el nombre del campo de seleccion. < select name="deporte"> < option >Fútbol< /option >< option >Críquet< /option >< option >Básquetbol< /option >< option >Hockey< /option >< option >Tenis< /option >< /select >
< select size="" > Indicara el numero de opciones visibles. Si le asignamos 1, la seleccion se presentara como un menu desplegable. Si le asignamos un valor mayor se presentara como una lista con barra de desplazamiento. < select size = "2">< option value="bca">BCA< /option>< option value="bcom">B.COM< /option>< option value="btech">B.TECH< /option>< option value="btech">B.SC< /option>< /select >
< select multiple > Indica si se pueden realizar multiples selecciones. < select name="bebidas" multiple>< option value="1">Coñac< /option>< option value="2">Vodka< /option>< option value="3">Whisky< /option>< option value="4">Ron< /option>< option value="5">Cerveza< /option>< option value="6">Ginebra< /option>< /select >
< textarea name="" > En esta etiqueta se introduce el nombre del campo. < TEXTAREA NAME="Texto" >
< textarea cols= > En esta etiqueta se introduce el numero de columnas de texto visible. < TEXTAREA COLS=20 >
< textarea rows= > En esta etiqueta se introduce el numero de filas de texto visible. < TEXTAREA ROWS=10 >
< input type="radio" > Se utiliza cuando se envía un formulario, ya que con este solo se envían al servidor los botones de radio que están marcados actualmente, y el valor informado es el valor del atributo de valor. Si el valor no se especifica de otra manera, es la cadena activada de forma predeterminada. < input type="radio" id="contactChoice2" value="phone" >
